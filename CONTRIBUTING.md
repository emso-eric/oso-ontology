# Contributing to the OSO Ontology

Thank you for your interest in contributing to the **Observatories of Seas Ontology (OSO)**.

OSO is developed collaboratively within the **EMSO Data Management Service Group (DMSG)** to support semantic interoperability and FAIR data management for marine observatories.

---

## Contribution Policy

Any contribution — including **updates, additions, or deletions** — must be reviewed and approved by the **EMSO Data Management Service Group (DMSG)** in order to maintain the quality, consistency, and interoperability of the ontology.

Contributions may include:

- new classes or properties
- corrections or improvements to existing concepts
- alignment with external ontologies
- metadata improvements
- documentation updates

---

## How to Contribute

You can contribute by following these steps:

1. **Open a GitHub issue** describing the proposed change.
2. Discuss the proposal with the OSO maintainers if needed.
3. Submit a **pull request** once the change has been agreed.

All contributions are reviewed by the **EMSO Data Management Service Group (DMSG)**.

---

## Commit Convention

This repository follows the **Conventional Commits** specification.

Examples:

feat: add Organisation alignment with CERIF  
fix: correct domain of oso:containsSite  
docs: update ontology metadata  
refactor: reorganise platform hierarchy  

Using this convention helps improve **traceability and changelog generation**.

---

## Versioning

OSO follows the **Semantic Versioning (SemVer)** specification.

Version format:

MAJOR.MINOR.PATCH

Meaning:

- **MAJOR** – incompatible ontology changes  
- **MINOR** – backward-compatible additions  
- **PATCH** – corrections, metadata improvements, or documentation updates
